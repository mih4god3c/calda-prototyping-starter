# Design Guidelines

This folder contains UX and UI patterns for CALDA prototypes. The AI reads these files to make consistent design decisions across screens.

## Files

| File | What it covers |
|------|---------------|
| `ux-patterns.md` | Navigation, loading states, empty states, forms, error handling, feedback |
| `component-patterns.md` | When to use which component — cards vs lists, modals vs drawers, etc. |

## How to Use

These are **defaults** — starting points that the AI will follow unless you tell it otherwise. You can:

- **Override a pattern** per screen: "Use a bottom sheet instead of a dialog for this action"
- **Add a new pattern**: Add a new section to the relevant file and commit it
- **Customise existing patterns**: Edit any section to match your product's conventions

## Contributing Patterns

When you discover a pattern that works well in your prototype, add it here so all future screens follow it. This makes the prototype feel more consistent as it grows.
